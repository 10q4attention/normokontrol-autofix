## Идея

Программа препода только **проверяет** и выдаёт отчёт об ошибках. Мы добавляем режим **«Проверить и исправить»**: система находит нарушения и сама правит `.docx`, сохраняя оригинал как резервную копию.

## Важные технические моменты

### Как python-docx правит форматирование
```python
# Поля страницы
from docx.shared import Mm
section = doc.sections[0]
section.top_margin    = Mm(20)
section.bottom_margin = Mm(20)
section.left_margin   = Mm(30)
section.right_margin  = Mm(15)

# Шрифт и размер в run
for run in paragraph.runs:
    run.font.name = 'Times New Roman'
    run.font.size = Pt(14)

# Межстрочный интервал
from docx.enum.text import WD_LINE_SPACING
paragraph.paragraph_format.line_spacing_rule = WD_LINE_SPACING.MULTIPLE
paragraph.paragraph_format.line_spacing = 1.5

# Замена кавычек
for run in paragraph.runs:
    run.text = run.text.replace('"', '«').replace('"', '»')
```

### Стратегия резервного копирования
```python
import shutil, datetime
backup = path.with_name(f"{path.stem}_backup_{datetime.datetime.now():%Y%m%d_%H%M%S}.docx")
shutil.copy2(path, backup)
```

## Что НЕ будем автоматизировать

- Добавление отсутствующих разделов (аннотация, введение и т.д.) — требует написания текста
- Исправление оглавления — слишком рискованно менять без понимания структуры
- Грамотность текста
