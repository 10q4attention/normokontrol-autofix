# Нормоконтроль ВКР — Автоматическое исправление

Расширение проекта [normokontrol-vkr-astafev](https://github.com/astafiev-rustam/normokontrol-vkr-astafev).

Оригинал только **проверяет** оформление. Этот форк умеет ещё и **автоматически исправлять** найденные нарушения: поля, шрифт, межстрочный интервал, кавычки, заголовки, подписи к таблицам и рисункам.

Перед исправлением всегда создаётся резервная копия оригинала.

---

> Система выполняет **формальную** проверку документа. После автоматической проверки преподаватель **обязан** проверить работу вручную — особенно позиции, которые система не проверяет.

---

## Быстрый старт

**1. Требования:** Python 3.10+

**2. Установка:**
```bash
pip install -r requirements.txt
```

**3. Запуск:**
```bash
python app.py
```

**4. Откройте** `http://localhost:5000`

**5. Укажите путь** к папке с работами и нажмите «Проверить».

---

## Структура папки с работами

```
Корневая папка/
├── Иванов Иван_данные/
│   └── 090302_22И0012_Иванов.docx
├── Петров Пётр_12345/
│   └── 090302_22П0034_Петров.docx
└── ...
```

Каждая подпапка = один студент. Система берёт последний по времени `.docx` или `.doc`.

## Что исправляется автоматически

При нажатии «Исправить» программа правит `.docx` и сохраняет резервную копию. Вот что меняется:

| Область | Что именно исправляет |
|---------|----------------------|
| **Поля страницы** | Устанавливает 30-20-20-15 мм (левое-правое-верхнее-нижнее), A4, книжная |
| **Основной текст** | Times New Roman 14pt, выравнивание по ширине, отступ первой строки 1.25 см, межстрочный 1.5, интервалы до/после 0 |
| **Заголовки** | Times New Roman 18/16/14pt, полужирный, убирает точку в конце, ПРОПИСНЫЕ для основных разделов, правильные интервалы и выравнивание по уровню |
| **Уровни заголовков** | Приводит уровни Heading к правильным: «1. Глава» → Heading 1, «1.1 Подраздел» → Heading 2, «1.1.1» → Heading 3; ВВЕДЕНИЕ/ЗАКЛЮЧЕНИЕ/… → Heading 1 |
| **Оглавление** | «Содержание» → «ОГЛАВЛЕНИЕ» (Heading 1); ПРОПИСНЫЕ названия основных разделов в тексте оглавления |
| **Кавычки** | Прямые `"..."` → «ёлочки» в русском тексте |
| **Подписи таблиц** | Times New Roman 12pt курсив, по ширине, интервал перед 6pt, дефис → тире (—) |
| **Подписи рисунков** | Times New Roman 12pt полужирный, по центру, интервал после 6pt, дефис → тире (—) |
| **Маркированные списки** | Times New Roman 14pt, выравнивание по ширине |
| **Нумерованные списки** | Times New Roman 14pt, выравнивание по ширине |
| **Листинги** | Код: Courier New 10pt; подпись листинга: Times New Roman 12pt курсив |

## Что придётся исправить вручную

Программа **не может** автоматически исправить следующее — это нужно сделать самостоятельно в Word:

| Область | Что нужно сделать вручную | Причина |
|---------|--------------------------|---------|
| **Структура ВКР** | Добавить отсутствующие разделы (Аннотация, Введение, Заключение и т.д.) | Нужно писать содержание |
| **Имя файла** | Переименовать по шаблону `XXXXXX_ГГГрГГГ_Фамилия.docx` | Зависит от данных студента |
| **Таблицы без подписей** | Добавить «Таблица X.Y — Название» перед таблицей | Неизвестно, как должна называться таблица |
| **Ссылки на рисунки** | Добавить в текст «(см. рисунок X.Y)» до появления рисунка | Неизвестно, в какое место вставить |
| **Список источников** | Добавить ссылки `[N]` в текст работы | Нужен содержательный анализ |
| **Формулы** | Набрать в редакторе Equation, добавить нумерацию, пояснение «где» | Риск повреждения объектов OLE |
| **Текст после таблиц/рисунков** | Добавить вводный текст перед крупными объектами | Нужно писать содержание |
| **Оглавление (заполнитель)** | Убедиться, что строки заканчиваются точками и номером страницы | Зависит от версии Word |

## Что проверяется автоматически

| № | Правило | Проверяется | Не проверяется (нужно вручную) |
|---|---------|------------|-------------------------------|
| 01 | **Структура ВКР** | Имя файла по шаблону, наличие обязательных разделов, порядок, нумерация | Содержание разделов, оформление титульного листа и задания |
| 02 | **Макет страницы** | Поля 20-20-30-15 мм, книжная ориентация, A4 | Колонки |
| 03 | **Правила текста** | Кавычки-ёлочки для русского текста, двухуровневая нумерация объектов, ссылки с большой буквы, текст после таблиц/рисунков | Пробелы после пунктуации, переносы слов, дефис/тире, сокращения, падежные окончания, имена собственные, расположение объектов после упоминания |
| 04 | **Основной текст** | Times New Roman 14pt, выравнивание по ширине, отступ 1.25 см, межстрочный 1.5, интервалы 0 | Цвет текста, подчёркивание |
| 05 | **Заголовки** | Times New Roman, полужирный, 18/16/14 пт, выравнивание, интервалы, ПРОПИСНЫЕ для основных разделов, без точки в конце | Цвет, положение на странице |
| 06 | **Оглавление** | Наличие, форматирование без отступа, ПРОПИСНЫЕ заголовки в содержании | Заполнитель точками, наезды строк |
| 07 | **Таблицы** | Подписи (Times New Roman 12pt курсив, по ширине, интервал перед 6pt), содержимое (12pt, по ширине/центру), заголовки столбцов по центру, строк по левому, графа «№ п/п», нумерация, ссылки, «Продолжение Таблицы» | Автоподбор, диагональное деление, интервал после таблицы |
| 08 | **Рисунки** | Подписи (Times New Roman 12pt полужирный, по центру, интервал после 6pt), нумерация, ссылки, без сокращения «Рис.», без трёхуровневой нумерации | Расположение после упоминания, удобство рассматривания без поворота |
| 09 | **Формулы** | Нумерация (X.Y), ссылки в тексте, пояснение со слова «где» без двоеточия и отступа | Редактор формул, пустые строки до/после, выравнивание, переносы |
| 10 | **Маркированные списки** | Маркер — тире (—), строчная буква, точка с запятой (последний — точка), Times New Roman 14pt, по ширине, отступы | Списки с неопределённым маркером |
| 11 | **Нумерованные списки** | Точка после номера, прописная буква, точка в конце, Times New Roman 14pt, по ширине, отступы | Списки с неопределённым маркером |
| 12 | **Листинги** | Подписи (Times New Roman 12pt курсив, по левому), код (Courier New 10pt, по левому), рамка, нумерация, ссылки | Ссылки на строки кода, белый фон |
| 13 | **Список источников** | Нумерация, Times New Roman 14pt, по ширине, отступ 1.25, ссылки в квадратных скобках | Правильность оформления записей по ГОСТ, порядок упоминания |
| 14 | **Приложения** | Буквы (кроме Ё,З,Й,О,Ч,Ь,Ы,Ъ), порядок, нумерация объектов, ссылки в тексте, заголовок по центру, интервалы 0 | Перечень на первом листе, содержимое |

## Что проверять вручную

После автоматической проверки **обязательно проверьте:**

1. **Титульный лист и задание** — соответствие шаблону СМКО МИРЭА
2. **Аннотация** — объём не более 150 слов, наличие ключевых слов (5-15)
3. **Объём работы** — 40-60 страниц основного текста (без приложений), приложения не более 1/3
4. **Содержание разделов** — соответствие этапам методички (литобзор, проектное обоснование, реализация)
5. **Правила текста** — пробелы после пунктуации, тире/дефисы, отсутствие переносов, падежные окончания числительных
6. **Оглавление** — заполнитель точками, нет наездов на номера страниц
7. **Формулы** — набраны в редакторе Equation, переносы после знаков, пустые строки
8. **Списки** — для списков с неопределённым маркером проверьте визуально
9. **Листинги** — белый фон, ссылки на строки кода
10. **Источники** — правильность записей по ГОСТ, порядок упоминания

## Ложные срабатывания

Система может ошибаться в случаях:
- Нестандартные стили (не Heading 1/2/3)
- Таблицы как изображения
- Сложное наследование стилей
- Автоматические списки с неопределённым маркером

**Всегда доверяйте визуальной проверке.**

## Добавление правил

Создайте файл `rules/rule_NN_name.py`:

```python
from .base_rule import BaseRule, RuleResult

class MyRule(BaseRule):
    @property
    def rule_id(self): return "my_rule"
    @property
    def name(self): return "Название"
    @property
    def description(self): return "Описание"

    def check(self, model):
        # model.elements, model.headings, model.tables, ...
        return RuleResult(status='pass', summary='OK')
```

Нажмите «Обновить правила» в интерфейсе.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Сервер не запускается | `pip install flask python-docx lxml` |
| Не видит файлы | Проверьте путь, наличие docx в подпапках |
| Ошибка в отчёте | Перезапустите сервер, проверьте консоль |
| Долгая проверка | Уменьшите количество проверяемых работ за раз |
